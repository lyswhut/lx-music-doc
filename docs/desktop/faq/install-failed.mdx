### Windows 安装版安装问题

#### 覆盖安装时安装失败

出现这个问题的原因一般是：你当初在安装本软件的时候是以「管理员身份」安装的，但在运行软件的时候「没有以管理员身份运行」，导致卸载后权限冲突无法再装上。

安装本软件时建议选择「**仅为我安装**」，并安装在「当前用户目录」或不需要管理员权限的目录（如其他分区下），不要选择「为使用这台电脑的任何人安装」。

#### 提示安装程序并未成功地运行完成或安装过程中出现错误

对于部分电脑出现安装失败的问题，可以做出以下尝试：

- 若之前可以安装成功，但现在安装失败，可以去「**控制面板 → 程序和功能/卸载程序**」或使用第三方卸载工具查看是否有之前的版本残留。若在不同路径下同时安装了多个版本，可能会出现该问题，卸载掉所有版本重新安装即可。
- 清理安装路径下的残留文件。
- 清理注册表（建议用清理工具清理）。

#### 安装结束后，无法启动软件

表现为安装结束后，安装目录内只有卸载程序，而没有其他程序文件。

- 可能是你使用了错误的安装包。
  - *例如在 x64 系统上使用了 Windows Arm64 版安装包就会出现这个问题。*
- 覆盖安装时更改了安装目录。
  - *覆盖安装时不要更改安装目录。若想更改目录，需将软件卸载再安装。*

解决方法：在「**控制面板 → 程序和功能/卸载程序**」中卸载软件，然后选择对应的正确安装包安装，一般即可解决。

#### 提示缺少 `xxx.dll`

如果你使用的是**免安装版**（压缩包），则需要将所有文件解压出来才能运行，否则将提示缺失 `ffmpeg.dll` 等文件。

如果你使用的是**安装版**出现这个提示，则可能是你电脑缺少某些系统组件导致的（正常的系统不会出现该问题）。可以尝试如下几个解决办法：

- 尝试重新安装软件。
- 以管理员身份打开命令提示符，输入 `sfc /scannow` 并回车，等待检查完成后重启电脑。
- 若 `sfc /scannow` 修复、重启电脑后仍然不行，请自行搜索弹出的**错误信息**具体内容，参考网络上的解决方案。

### macOS 无法启动软件，提示 `lx-music-desktop 已损坏`、`无法打开“lx-music-desktop”`、`无法验证开发者`

这是因为软件没有签名，系统安全机制阻止了程序运行。

在终端里输入以下命令并回车，然后输入你的电脑密码（输入密码时屏幕不会显示字符）即可：

`sudo xattr -rd com.apple.quarantine /Applications/lx-music-desktop.app`
